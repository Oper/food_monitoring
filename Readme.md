## Описание
Проект родился для автоматизации рабочих процессов современной Школы (а также оконченной эпидемии Ковида, но все еще актуальных отчетов). Проект (на данный момент) позволяет развернуть небольшой сервис на локальном сервере (и не только локальном).
Проект создавался под свои нужды, функционал добавляю по наличию времени и идей для реализации конкретного набора функции.

## Возможности
*   Сбор информации болеющих по классам и передавать данные по электронной почте;
*   Оформления Школьного меню (создание блюд и меню на каждый день).
*   Выгрузка меню для портала мониторингпитания.рф

## Установка
Пошаговая инструкция о том, как запустить проект локально (Docker).

**Предварительные требования (Prerequisites):**
*   [Python](www.python.org) (версия 3.11+)
*   [Git](git-scm.com)
*   [PostgreSQL](www.postgresql.org/)

**Инструкции:**

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Oper/food_monitoring.git
   cd food_monitoring/app
2. Поднимаем базу на PostgreSQL
3. Создаем в папке "app" файл .env с параметрами:
   ```
   DB_HOST=
   DB_PORT=
   DB_NAME=
   DB_USER=
   DB_PASS=
   SECRET_KEY=
   ALGORITHM=HS256
   HOST_MAIL=
   PORT_MAIL=
   USER_MAIL=
   PASSWORD_MAIL=
   TO_MAIL=
   SCHOOL='Наименование школы' 
4. Собираем Docker образ
   ```bash
   docker build -t name_image .
   
5. Запуск контейнера:
   ```bash
   docker run -d -e TZ=Europe/Moscow -p 80:80 name_image
